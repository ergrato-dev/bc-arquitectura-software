|--------|---------|-------------------|

### Complementarios

| Título | Autores | Notas |
|--------|---------|-------|
| **Design Patterns: Elements of Reusable OO Software** | Gang of Four | El libro original. Ejemplos en Smalltalk/C++ pero los conceptos aplican a todo |
| **Clean Code** | Robert C. Martin | Contexto sobre qué hace que un código sea legible y mantenible |
| **Refactoring** | Martin Fowler | Cómo mejorar código existente — directamente relacionado con refactorizar hacia patrones |

## 🛠️ Herramientas de Desarrollo

Para esta semana necesitas:

```bash
# Node.js 20+ (LTS) — soporte completo ES2023
node --version  # >= 20.x

# pnpm (gestor de paquetes del bootcamp)
pnpm --version  # >= 8.x

# Para tests (opcional pero recomendado)
node --test     # Test runner nativo de Node.js (v18+)
```

### Extensiones VSCode Recomendadas

- **ESLint** — Análisis estático de código
- **Prettier** — Formateo consistente
- **PlantUML** — Preview de diagramas UML en VSCode
- **Mermaid Preview** — Preview de diagramas Mermaid
- **GitHub Copilot** — Apoyo en la implementación (¡pero entiende primero el patrón!)
